## 決定事項

- フレームワークは Next.js App Router
- API は `app/api/*` の Route Handler で提供する
- `RawVideoData` 互換を維持しつつ、追加情報は `diagnostics` で返す
- Pencil の初期デザイン案は `docs/sample/youtube-research-console.pen` に保存する
- Pencil の横並び微調整では、右パネル見出し高さを左パネルに合わせ、デバッグ用ラベルは残さない
- Pencil のURL一覧は URL 単位で直接編集・追加できる見た目にし、右パネルへ送る補助ボタン `抽出へ送る` を置く
- 左パネル `0e3Pg` のURL一覧は、個別カードではなく改行区切りでまとめて編集できる単一テキストボックス形式にする
- 右パネル `qXHU6` には `コメントのみ` ボタンを追加し、コメントだけを抽出できるようにする
- 右パネル `qXHU6` の入力欄は複数URLを改行区切りで貼り付けられるようにし、抽出結果はURLごとのカードを縦並びで表示する
- 複数URL抽出では、ボタン押下直後にURLカードを先出しし、各カードで `抽出中` / `完了` / `失敗` を個別表示する
- 抽出結果カードは初期表示をタイトル+URLの要約だけにし、開いた時だけ詳細を確認できる形にする
- 字幕取得率を上げるため、caption track の `fmt=json3` フォールバックを追加する
- `yt-dlp` が Vercel でも動くように、公式バイナリを install 時に取得して serverless trace に含める
- 字幕抽出 API は Vercel で `hnd1` を優先リージョンに設定し、日本リージョン寄りで実行する
- YouTube 側の bot check 用に、必要なら `yt-dlp` へ cookies を渡せる env 受け口を用意する
- 本番環境で字幕が取りこぼれにくいよう、`youtube-transcript-plus` / watch page / Innertube の字幕系リクエストは browser-like header を付け、必要時は別 desktop User-Agent でも再試行する
- caption track が `signatureCipher` / `cipher` 形式で返るケースも拾えるように URL 復元を入れる
- サービスアカウントJSONを使って `AI抽出` シートへ反映するボタンを実装し、抽出成功分を所定の列へ追記できるようにする
- Sheets 連携で使う環境変数は、前後の空白や改行が混じっていても trim して扱う
- 台本本文は `AI抽出` に直書きせず、同一スプレッドシート内の `台本DB` シートへ chunk 分割で保存し、`/scripts/[scriptId]` で閲覧できるようにする
- `AI抽出` はヘッダー行を持つ一覧シートとし、右端に `台本` / `コメント` 列を置き、閲覧ページから JSON ダウンロードもできるようにする
- 字幕抽出は最初に取れた結果で確定せず、より末尾まで取れている候補を優先する
- 字幕表示はプレビュー件数で切らず、取得できた全件をスクロール領域内で表示する
- 実装UIは `docs/sample/youtube-research-console.pen` の最新構成を基準にし、左で取得したURL一覧を編集して右の抽出欄へ送れる構成にする
- `/` はサーバー側で `docs/sample/youtube-research-console.pen` を読み、主要文言・基調色・主要カード高さ・主要ボタン幅を UI に反映する
- サンプルUIの文言は日本語を基本とし、「URLを解決」のような意味が伝わりにくい表現は避ける
- 保存や分析連携は v1 に含めない
