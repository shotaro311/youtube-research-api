## 決定事項

- フレームワークは Next.js App Router
- API は `app/api/*` の Route Handler で提供する
- `RawVideoData` 互換を維持しつつ、追加情報は `diagnostics` で返す
- Pencil の初期デザイン案は `docs/sample/youtube-research-console.pen` に保存する
- Pencil の横並び微調整では、右パネル見出し高さを左パネルに合わせ、デバッグ用ラベルは残さない
- Pencil のURL一覧は URL 単位で直接編集・追加できる見た目にし、右パネルへ送る補助ボタン `抽出へ送る` を置く
- 左パネル `0e3Pg` のURL一覧は、個別カードではなく改行区切りでまとめて編集できる単一テキストボックス形式にする
- 右パネル `qXHU6` には `コメントのみ` ボタンを追加し、コメントだけを抽出できるようにする
- 右パネル `qXHU6` の入力欄は複数URLを改行区切りで貼り付けられるようにし、抽出結果はURLごとのカードを縦並びで表示する
- 複数URL抽出では、ボタン押下直後にURLカードを先出しし、各カードで `抽出中` / `完了` / `失敗` を個別表示する
- 抽出結果カードは初期表示をタイトル+URLの要約だけにし、開いた時だけ詳細を確認できる形にする
- 字幕取得率を上げるため、caption track の `fmt=json3` フォールバックを追加する
- `yt-dlp` が Vercel でも動くように、公式バイナリを install 時に取得して serverless trace に含める
- 字幕抽出 API は Vercel で `hnd1` を優先リージョンに設定し、日本リージョン寄りで実行する
- YouTube 側の bot check 用に、必要なら `yt-dlp` へ cookies を渡せる env 受け口を用意する
- 本番環境で字幕が取りこぼれにくいよう、`youtube-transcript-plus` / watch page / Innertube の字幕系リクエストは browser-like header を付け、必要時は別 desktop User-Agent でも再試行する
- caption track が `signatureCipher` / `cipher` 形式で返るケースも拾えるように URL 復元を入れる
- サービスアカウントJSONを使って `動画分析` シートへ反映するボタンを実装し、抽出成功分を所定の列へ追記できるようにする
- Sheets 連携で使う環境変数は、前後の空白や改行が混じっていても trim して扱い、旧名 `AI抽出` / `コメントシート` が設定されていても新名へ読み替える
- 台本本文は `動画分析` に直書きせず、同一スプレッドシート内の `台本DB` シートへ chunk 分割で保存し、`/scripts/[scriptId]` で閲覧できるようにする
- `動画分析` はヘッダー行を持つ一覧シートとし、右端に `台本` / `コメント` 列を置き、閲覧ページから JSON ダウンロードもできるようにする
- `動画分析` のサムネは約4倍サイズのカスタム表示にし、追加行の行高とサムネ列幅も合わせて広げる
- `/scripts/[scriptId]` では大きめのサムネを表示し、クリックで元画像を開けるようにする
- `/scripts/[scriptId]?tab=comments` では投稿者名と本文を分けたカード表示にする
- `/scripts/[scriptId]?tab=transcript` では `台本分析` ボタンで、元の台本の右側に `動画の流れ / 視聴者心理の変化 / 寄り添い発言 / 企画意図 / 良い部分 / 改善が必要な部分 / 総合評価` を表示できるようにする
- 台本分析は台本本文とメタデータをもとに行い、総合評価では再生数・登録者数・コメント数も考慮する
- 抽出結果カードでも `分析` ボタンから `台本分析 / コメント分析` を実行でき、同じカードの下側でタブ切り替え表示できるようにする
- `/scripts/[scriptId]` の台本タブとコメントタブには、それぞれ一括コピーボタンを置く
- `台本分析` のモデルは `gemini-3.1-flash-lite-preview` を使う
- `/scripts/[scriptId]?tab=comments` では `コメント分析` ボタンから、上部に折りたたみ可能な総評、下部にコメント本文を強調した個別フィードバック一覧を表示できるようにする
- コメント分析結果は、分析一式のコピーと JSON 保存に対応させる
- `/scripts/[scriptId]` のタブ切り替えは client 側で行い、台本 / コメントの往復をよりスムーズにする
- コメント分析結果は `新着順` と感情タグのフィルターで見たい反応だけ追えるようにする
- コメント分析の比率表示は件数ベースで必ず一致させる
- `台本分析` と `コメント分析` は同じ `scriptId` で再読込しても表示を復元する
- `コメント分析` のモデルは `gemini-3.1-flash-lite-preview` を使う
- コメント分析は、総評ブロックと個別コメントの両方を編集して保存できるようにし、総評は `台本DB` の `comment_analysis` 行へ、各コメントの評価情報は `コメントDB` と `コメント分析` の各行へ同期する
- コメント分析で感情タグを変えた場合は、その場で比率カードとフィルター件数も再計算して表示を更新する
- コメント分析の比率カードとフィルターは、アイコン付きでクリックするとコメント一覧へ移動しつつ切り替わる形にする
- コメント分析総評には削除ボタンを追加し、保存済みの分析結果を `台本DB` から消せるようにする
- コメント分析の保存導線はカード上部の操作列に寄せ、感情タグだけは編集モード外でも直接変更できるようにする
- `scripts/[scriptId]` にはルート専用のローディングUIを追加し、シートから開いた直後の白画面を避ける
- Sheets 反映時は viewer 用の `台本DB` に加えて、分析用の `コメントDB` と閲覧用の `コメント分析` にも `1コメント = 1行` で保存する
- `コメントDB` と `コメント分析` が未作成でも初回反映時にヘッダー付きで自動作成できるようにする
- `コメントDB` は raw コメントだけでなく、コメント分析で編集した `sentiment / viewer_type / psychology / note / analysis_updated_at` も保持する
- `コメント分析` は `動画タイトル / 動画リンク / チャンネル名 / コメント本文 / 感情タグ / 視聴者像 / 心理 / 個別フィードバック` を中心に、人が一覧しやすい形で保持する
- `動画分析` へ新しい動画を反映する時点で、同一動画の既存分析済みコメントが `コメントDB` にあれば、その分析列を `コメントDB` と `コメント分析` の新規行へ自動反映する
- 現在の `台本DB` は 279 行で正常に読めており容量面は十分余裕があるが、実装は `台本DB!A:H` の全件読みに依存しているため、長期的には件数増加時の読み出し遅延に注意する
- 字幕抽出は最初に取れた結果で確定せず、より末尾まで取れている候補を優先する
- 字幕表示はプレビュー件数で切らず、取得できた全件をスクロール領域内で表示する
- 実装UIは `docs/sample/youtube-research-console.pen` の最新構成を基準にし、左で取得したURL一覧を編集して右の抽出欄へ送れる構成にする
- `/` はサーバー側で `docs/sample/youtube-research-console.pen` を読み、主要文言・基調色・主要カード高さ・主要ボタン幅を UI に反映する
- サンプルUIの文言は日本語を基本とし、「URLを解決」のような意味が伝わりにくい表現は避ける
- 左右パネルとも主要アクションボタンは入力URL欄の上に配置する
- 保存や分析連携は viewer と Sheets 連携の範囲で v1 に含める
